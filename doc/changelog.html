Changelog

This is the changelog of couchbeam.

0.4.3 - July, 14, 2010

  • New build system using rebar
  • Added R14a support
  • Fix tests
  • Fix issue #29, make sure change line is handled in change consumer
  • Fix issue #28, make sure ID is a binary when we need to create it for a new doc.

0.4.1 - November, 22, 2009

  • Added couchbeam_view:first/1 and couchbeam_view:first/2 function allowing you to retrieve first result of a view.
  • Fix. Key in params could now be an atom, binary or list.
  • Fix. encode dbname
  • Support for couchdb-lucene or other external indexer in views results.
  • include_docs is now handled in view results. now fetch_view return
    • [id] if value not given
    • {Id, Key, Value] -> if value only
    • {Id, key, Value, Doc} if doc is included.
  • Make sure docid is encoded.
  • Fix. couchbdeam_doc:extend update property if it exists or add it.

0.4 - October, 24, 2009

  • Added support for attachments streaming
  • Added couchbeam_doc:delete_value/2 function
  • Added some shorcuts functions :
    • get_id/1
    • get_rev/1
    • get_idrev/1
    • is_saved/1
  • fix docid encoding. Now support special chars
  • fix couchbeam_doc:get_value/2 and added couchbeam_doc:get_value/3 to define default value
  • Some code refactoring in couchbeam_resource module to support streaming.
  • doc cleanup

0.3 - October, 16, 2009

  • suscribe to db changeswith couchbeam_db:suscribe/2, couchbeam_db:suscribe/3 , refer to doc for more information.
  • capabilities to recover server connections and dbs in case something bad happened.
  • couchbeam_server:close(Connection) has been added.
  • couchbeam_db:register_db don't exist anymore. Instead use couchbeam_db:open_db(Connection, {Alias, DbName}), or just couchbeam_db:open_db(Connection, DbName), then alias of db is {ConnectionName, DbName}.
  • couchbeam_manager:register_server, parameters changed it take now ConnectionPid, CouchdbParams.
  • fix view closing which sometimes restarted the db gen_server

0.2 - September, 30, 2009

  • complete rewrite of API. full OTP application. More details of new API are here
  • use of lhttpc rather than our own client. lhttpc is more tested than old http client also it allows us to reuse connections.
